Exercises (Version Control Systems)

Task 1 (10 points). Dice Game

  1. Create a new Git repository.

  2. Use any programming language to write a simple “Dice Game” application. The program imitates rolling two dice by generating and printing two random numbers (from 1 to 6) on the screen:

    Rolling dice...
Die 1: 5
Die 2: 3
Total value: 8

  3. Commit this program as your first revision with the message “Dice game: initial revision”.

  4. Create a separate branch called experimental. Modify the program in this branch so it asks the user to provide a name first, and then greets the user as follows:

    What is your name? 
> John
Hello, John!

    After greeting the user, the program imitates rolling the dice exactly as in the previous version. Commit this new revision with the message “Greetings added”.

  5. Add the following code to the the master branch: if the total dice value is greater than 7, the program prints the message You won, otherwise it prints the message You lost. Commit this revision with the message “Won/lost message added”.

  6. Merge experimental branch into the master branch. Change the code so it prints the user name instead of You. Thus, the resulting dialog would look like this:

    What is your name? 
> John
Hello, John!
Rolling the dice...
Die 1: 5
Die 2: 3
Total value: 8
John won!

  7. Commit this new revision with the message “Won/lost message modified”. Your current branch should be master.
